Object Oriented Programming

  • OOP is the style of programming that allows us create objects with properties and methods that act on the properties.
  • An instance of a class is created to use it.
  • JavaScript didn't have classes before ES6
  • The "this" keyword refers to an instance of the class.
  • All code in a class is implicitly in strict mode
  • An error is thrown when trying to access a class that was created using the class declaration method without the "new" keyword"
  • Static method are called directly using the class name and not an instance of the class.
  • dunder proto is short for "double underscore proto" // __proto__
  • never use arrays or objects as a default value in prototype.

mixins

  • This is a way of adding properties and methods of some objects to another object without using inheritance.
  • mixin is achieved through the use of Object.assign() method.

Composition over inheritance

  • Gorilla Banana is the situation where you need a method from a class but end up inheriting the whole class to gain access to the particular method.
  • The composition over inheritance solves the above problem.
