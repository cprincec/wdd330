Notes on OBJECTS LITERALS

  • JavaScript Object data type that stores data in key-value pairs.
  • Data is not arranged in any particular order.
  • Data can be accessed using the dot notation.
  • Can also store functions.
  • Functions stored in objects literals are known as methods.
  • hadOwnProperty() is used to to know if a property exists in an object literal. This is not include inhherited properties.
  • Object.entries(objectName) returns the keys and objects in an object literal.
  • delete can be used to remove an item from an object.

JSON

  • JavaScript Object Notation is a light-weight data storage format used to store large number of services/data. invented by douglas Crockford in 2001.
  • Different from Object literal
  • property names are in double quotes.
  • functions are not permited values.
  • Has the parse() method that converts the JSON strings to JavaScript Object.
  • the parse() strips the keys of their double quotes.
  • stringify converts JavaScript Object to JSON data.
  • Math.trunc() returns only the integer parts of a number. Eg. Math.trunc(4.9) // returns 4.
  • Date Objects have getter and setter methods.
  • A day can be represented by 1000 * 60 * 60 * 24 milliseconds.
  • getTimezoneOffset() returns the difference between the local time on the computer and UTC.
